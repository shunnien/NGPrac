Introduction

使用 ngHref 的原因是避免 hyperlink 在angular.js 沒有載入成功或是還沒載入完的話 連結錯誤造成 404 的頁面

ngSrc 則是 angular.js 載入失敗時，不管是直接使用 src 或是改用 ng-src 都會顯示所謂的叉燒包圖片(Chrome 及 Firefox 遇到沒有 src 屬性的圖片會不顯示)
使用 ngSrc 的主要原因是元素的 src 基本上就是會跟隨著頁面文件的載入而一起被讀取，而 AngularJS 是當頁面文件內容載入後完才進行。因此若是直接使用 src 加上 Angular 表達式(Angular Expressions)的圖片路徑就會被請求載入一次了，但這是不必要且是錯誤的。
